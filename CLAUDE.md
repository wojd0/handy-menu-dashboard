# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Build & Run

This is a **macOS-only** SwiftUI menu bar app (not iOS). `SUPPORTED_PLATFORMS = macosx`.

- **Open in Xcode:** `open "handy-menu-dashboard.xcodeproj"` then Cmd+R
- **Build:** `make build`
- **Install while iterating:** `make install-dev` builds Release and replaces `~/Applications/handy-menu-dashboard.app`
- **Run tests:** `make test`
- **Lint:** `make lint` (requires swiftlint; non-fatal if missing)

Scheme name: `handy-menu-dashboard`

## Project Configuration

## Feature Flags

Feature flags live in `handy-menu-dashboard/FeatureFlags+Generated.swift`, which is auto-generated by `scripts/generate-feature-flags.sh` (called by every build/test script). To enable a flag, set the matching env var before building:

- **`SHOW_GITHUB_SETTINGS`** â€” shows the GitHub Copilot tab in Settings and Copilot card in Dashboard (default: `false`). Tests always set this to `true`.

You can also create a `.env` file in the project root; the script sources it automatically.

## Architecture

Menu bar app using `MenuBarExtra` with `.window` style showing coding agent usage statistics.

### Data Flow

Services (`CursorService`, `CopilotService`, `ClaudeService`) are `@Observable` classes created as `@State` in the `App` struct and passed as parameters to views â€” not via `@Environment`. All services auto-refresh every 3 minutes via background `Task` loops and store credentials in Keychain.

### Business Logic

**`UsageMath.swift`** contains all pure calculation functions (`spendPercentUsed`, `copilotPercentUsed`, `cursorMenuBarFragment`, `percentUsed`, `claudeDollarsFragment`, etc.) and the JSON response structs (`TeamSpendResponse`, `FilteredUsageEventsResponse`, `ClaudeUsageResponse`). This is the right place for numeric/formatting logic â€” keep it side-effect free. `FlexibleInt` handles Cursor's API returning integers as either numbers or strings.

### Services
- **CursorService** â€” Fetches Cursor IDE team spend via `cursor.com/api/dashboard/get-team-spend`. Cookie-based auth (WebView login flow). Extracts `team_id` from cookies, fetches team spend, then identifies the current user by matching email from `/api/auth/me`.
- **CopilotService** â€” Fetches GitHub Copilot premium request usage via GitHub API (`/users/{username}/settings/billing/premium_request/usage`). Fine-grained PAT auth. User-configurable monthly entitlement (default 300).
- **ClaudeService** â€” Fetches Claude.ai usage via `claude.ai/api/organizations/{orgId}/usage`. Cookie-based auth (WebView login). Org ID read from the `lastActiveOrg` cookie. Tracks two pools: **spend** (usage limit) and **cinder_cove** (Claude Code & Cowork credit). Supports three baselines (`usageLimit`, `credit`, `combined`) each with Dollars or Percent menu-bar display.
- **KeychainStoring** / **KeychainStore** â€” Protocol + production implementation wrapping macOS Security framework (`SecItem*` APIs). Injected into services; tests use `InMemoryKeychainStore`. All keys prefixed `com.wojd0.dashboard.*`. Includes automatic migration from legacy keychain format.

### Views
- **handy_menu_dashboardApp** â€” App entry point. `MenuBarExtra` shows usage summary; separate `Window("Settings", id: "settings")` opened via `openWindow`.
- **DashboardView** â€” Main popover with usage cards for each service, settings gear, and quit button. Copilot card is gated behind `FeatureFlags.showGitHubSettings`.
- **UsageCardView** â€” Reusable card showing service name, percentage, detail text, and progress bar.
- **UsageProgressBar** â€” Gradient progress bar (greenâ†’yellowâ†’orangeâ†’red based on usage).
- **SettingsView** â€” Settings window with per-service cards for Cursor (WebView login), Copilot (PAT entry + entitlement), and Claude (WebView login + baseline/unit pickers). Each service has its own Dollars/Percent menu-bar switch. Claude also has a 3-step baseline picker (Limit / Credit / Combined).
- **CursorLoginView** â€” `WKWebView` wrapper for Cursor OAuth login and cookie extraction.
- **ClaudeLoginView** â€” `WKWebView` wrapper for Claude login. Extracts `claude.ai` cookies after the user signs in and navigates away from `/login`.
